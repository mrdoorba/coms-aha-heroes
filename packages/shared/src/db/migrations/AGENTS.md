## For AI Agents

### Working In This Directory
- **NEVER manually edit** any `0NNN_*.sql` file generated by drizzle-kit. Editing them breaks the migration journal and will cause `drizzle-kit` to error on next generate.
- To introduce a schema change: edit the relevant file in `src/db/schema/`, then run `bun run db:generate`. A new numbered SQL file will be created automatically.
- `0002_triggers.sql` and `0003_rls_policies.sql` are hand-authored exceptions — they may be edited carefully if trigger or RLS logic needs updating, but prefer additive changes.
- To apply pending migrations to a running database: `bun run db:migrate`.

### Common Patterns
- New migrations always get the next sequential number (`0006_`, `0007_`, …).
- Do not rename or reorder existing migration files.

## Dependencies
